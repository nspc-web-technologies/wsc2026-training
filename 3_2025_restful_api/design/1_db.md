# Database Design - WSC Module C

## Table Structure

### 1. users

#### Fields

- `id` - Primary key (INT, AUTO_INCREMENT)
- `username` - Username (VARCHAR(255), UNIQUE)
- `email` - Email address (VARCHAR(255), UNIQUE)
- `password` - Hashed password (VARCHAR(255))
- `role` - User role (ENUM('admin','user'), default: 'user')
- `is_banned` - Ban flag (BOOLEAN, default: FALSE)
- `created_at` - Created at (TIMESTAMP)
- `updated_at` - Updated at (TIMESTAMP)

#### Constraints

- UNIQUE: `username`
- UNIQUE: `email`

### 2. access_tokens

#### Fields

- `id` - Primary key (INT, AUTO_INCREMENT)
- `user_id` - User ID (INT, foreign key)
- `token` - MD5 token (VARCHAR(32), UNIQUE)
- `created_at` - Created at (TIMESTAMP)

#### Constraints

- UNIQUE: `token`
- FOREIGN KEY: `user_id` â†’ `users(id)` (ON DELETE CASCADE)

#### Token logic

- Generated by: `md5($user->username)` (lowercase hex)
- On login: DELETE existing token by user_id â†’ INSERT new token
- On logout: DELETE token by user_id

### 3. albums

#### Fields

- `id` - Primary key (INT, AUTO_INCREMENT)
- `title` - Album title (VARCHAR(255))
- `artist` - Artist name (VARCHAR(255))
- `release_year` - Release year (INT)
- `genre` - Genre (VARCHAR(255))
- `description` - Description (TEXT)
- `publisher_id` - Publisher user ID (INT, foreign key)
- `deleted_at` - Soft delete timestamp (TIMESTAMP, NULLABLE)
- `created_at` - Created at (TIMESTAMP)
- `updated_at` - Updated at (TIMESTAMP)

#### Constraints

- FOREIGN KEY: `publisher_id` â†’ `users(id)` (ON DELETE RESTRICT)

### 4. songs

#### Fields

- `id` - Primary key (INT, AUTO_INCREMENT)
- `album_id` - Album ID (INT, foreign key)
- `title` - Song title (VARCHAR(255))
- `duration_seconds` - Duration in seconds (INT)
- `order` - Order within album (INT)
- `lyrics` - Lyrics (TEXT)
- `is_cover` - Is cover flag (BOOLEAN, default: FALSE)
- `cover_image_path` - Cover image path (VARCHAR(512), NULLABLE)
- `view_count` - View count (INT, default: 0)
- `deleted_at` - Soft delete timestamp (TIMESTAMP, NULLABLE)
- `created_at` - Created at (TIMESTAMP)
- `updated_at` - Updated at (TIMESTAMP)

#### Constraints

- FOREIGN KEY: `album_id` â†’ `albums(id)` (ON DELETE RESTRICT)

#### Cover image storage

- DB value (`cover_image_path`): `uploads/songs/{song_id}.jpg` (relative to public/)
- Physical path: `public/uploads/songs/{song_id}.jpg` (= `public_path('uploads/songs/{song_id}.jpg')`)
- URL: `/api/songs/{song_id}/cover`

### 5. labels

- `id` - Primary key (INT, AUTO_INCREMENT)

### 6. song_labels

#### Fields

- `id` - Primary key (INT, AUTO_INCREMENT)
- `song_id` - Song ID (INT, foreign key)
- `label_id` - Label ID (INT, foreign key)

#### Constraints

- UNIQUE: (`song_id`, `label_id`)
- FOREIGN KEY: `song_id` â†’ `songs(id)` (ON DELETE CASCADE)
- FOREIGN KEY: `label_id` â†’ `labels(id)` (ON DELETE CASCADE)

## Business Rules

1. Soft delete: `deleted_at` column for albums and songs (no physical delete)
2. Cascade soft delete: When album is soft-deleted, all its songs are soft-deleted
3. Token management: MD5(username) fixed value, DELETEâ†’INSERT on login
4. Label validation: Labels must exist in the labels table
5. Order management: New song order = MAX(order)+1 (including soft-deleted), no re-ordering after delete
6. Admin constraint: At least one admin must exist at all times
7. Cover image: DB stores `uploads/songs/{song_id}.jpg` (relative); physical path is `public/uploads/songs/{song_id}.jpg`; JPEG only
