## はじめに

このミニスピードテストプロジェクトでは、複数のミニタスクが与えられる。複数のクライアントからタスクを収集し、優先順位に基づいて今回の開発ラウンド用にいくつかのタスクを選定した。必要なタスクのみ実装すること。

どれを実装するかは指示に従うこと。

各ミニテストプロジェクトには3つのレベルがある。

- Level 1: 0.5ポイント。想定作業時間15分未満。
- Level 2: 1.0ポイント。想定作業時間30分未満。
- Level 3: 1.5ポイント。想定作業時間約30分。

ファイルは `/XX_module_a/` にタスクIDをフォルダ名として配置すること。例: `/XX_module_a/A1/`

XXはワークステーション番号。

## プロジェクトとタスクの説明

各ミニスピードテストプロジェクトへのリンクを含むインデックスページを作成すること。

インデックスページには各ミニスピードテストプロジェクトのサムネイルとタイトルを含めること。

各ミニスピードテストプロジェクトにつき少なくとも1つのgitコミットを書くこと。gitコミットメッセージは読みやすく理解しやすいものにすること。

## デザイン実装タスク

### A1: 拡大ドック (Level 1)

HTML、CSS、JavaScriptを使用して拡大ドックを作成する。マウスがドック上にホバーすると、周囲の要素が拡大される。最大距離340px、最大スケール1.3を参考値として使用できるが、より優れた効果を得るために自由に調整してよい。さらに、カーソルに追従するグロー効果をアイテム周囲に追加し、全体的なビジュアルの魅力を高めること。

要件ノート

- カーソルと共にボックス周囲を移動するグロー効果が追加されている。
- 拡大効果はボックスの中心で最大となる。
- カーソルに近いオブジェクトは拡大され、遠いものはマウス移動に伴い小さく表示される。
- 拡大は1つのボックスから別のボックスへ徐々に減少し、複数のボックスが同時に拡大される。
- マウスの移動が拡大効果のスムーズなトランジションにつながる。

### A2: ネオブルータリズムアニメーションボタン (Level 1)

HTMLとCSSのみを使用して、ホバー時にフローティング効果を表示し、クリック時に吊り下げられたボタンが押されるシミュレーションを表すアニメーションを表示するボタンを作成する。

期待される結果は `video-B1.mp4` ファイルに示されたものと同じであること。

要件ノート

- HTMLとCSSのみでボタンが実装されている。
- ホバー効果が実装されている。
- アクティブボタン効果が実装されている。
- クリックを離すとボタンが元の位置に戻る。

### A3: アニメーションリンク (Level 1)

HTMLとCSSのみを使用して、HoverとActiveの2つの異なる動作に対して2つの異なるアニメーションを持つアニメーションリンクを作成する。

期待される結果は `example-B2.mp4` ファイルに示されたものと同じであること。

要件ノート

- リンクのホバーアニメーションが実装されている。
- リンクのアクティブアニメーションが実装されている。
- JavaScriptを使用せずにアニメーションが実装されている。

### A4: ジャンピングボール (Level 2)

提供されたビデオに従い、3つの白いボールのモーション/シャドウ効果を再現する。協調的にアニメーションさせること。

期待される結果は `example-B3.mp4` ファイルに示されたものと同じであること。

要件ノート

- 提供されたデザインと同様のボールを再現した。
- ジャンプアニメーションを実装した。
- 提供されたビデオに従ったモーションとシャドウ効果のアニメーションがある。
- アニメーションが協調的に動作している。

### A5: オリンピックリング (Level 2)

HTMLとCSSのみを使用して、提供された画像を参考にオリンピックロゴをデザインする。

要件ノート

- オリンピックロゴの円が作成されている。
- 円のオーバーレイが提供されたメディアファイルと類似している。

### A6: ローディングアニメーション (Level 1)

ビデオを見て、スピナーローディングアニメーションを再現する。アニメーションは提供されたHTMLページのローディング中のみ実行されるべきである。テストにはブラウザをslow 3Gコネクションと6x slowdown CPUに設定する必要がある。

要件ノート

- ローディングアニメーションが作成されている。
- ローディングアニメーションが提供されたメディアファイルに従っている。
- ローディングアニメーションはページのローディング中のみ表示される。

### A7: カスタムカーソル (Level 2)

提供されたビデオと画像に従い、同じ効果のカスタムカーソルを再現する。

期待される結果は `example-C3.mp4` ファイルに示されたものと同じであること。

要件ノート

- ホバー要素が作成されている。
- ホバー要素が提供されたメディアファイルに示された形状に従っている。
- ホバー要素がメディアファイルで提供された効果を再現している。

### A8: グラスモーフィズムカード (Level 2)

提供されたHTMLページから、サンプルに示された入力ボタンを使用して、ファイル内のグラスモーフィズムカードのプロパティをユーザーが変更できるようにする。

カードのプロパティ変更に加えて、変更されたCSSコードをカードの下部に表示する必要がある。

参考用ビデオ (`example-C4.mp4`) が提供される。コントロール用のインプットを追加すること:

- 透明度 (opacity)
- ブラー
- カラー
- アウトラインの透明度

要件ノート

- インプットが実装されている (Transparency (range), Blur (range), Color (color), Outline (range))。
- カードがインプット値の変更に応じて変化する (動作しないインプットごとに0.15減点)。
- 提供されるCSSコードがページに実装されたものと同じ。

デスクトップとモバイルのレイアウトがリファレンス画像として提供される。

### A11: カラフルボタン (Level 3)

以下のHTMLが与えられる:

```html
<button id="btn">Button</button>
```

以下のCSSが与えられる:

```css
* { padding: 0; margin: 0; }
html, body { height: 100%; }
body {
    display: flex;
    justify-content: center;
    align-items: center;
}
:root {
    --bgc: #ff3366;
    --bgc2: #ff9977;
    --bgc3: #ffdd99;
    --bgc4: #ffff77;
}
#btn {
    position: relative;
    border-radius: 10px;
    overflow: hidden;
    width: 280px;
    height: 80px;
    margin: 30px;
    border: 1px solid;
    background-color: #fff;
}
```

CSSのみを使用し、HTMLを変更せずに以下のホバー効果を実装する。マウスがボタン上にホバーすると、4つのカラーバーがボタン内に表示され、左から右へ、上から下へ移動する。同時に、ボタンの対応するコーナーから4つのアークが出現し、サイズが拡大する。マウスが要素から離れると、アークは元の位置に戻り、4つのカラーバーは左から右へ、下から上へ移動して消える。

効果ビデオ `colorful-button.mp4` がメディアファイルで提供される。

要件ノート

- マウスがボタン上にホバーすると、4つのカラーバーがボタン内に表示され、左から右へ上から下へ移動する。CSSのみ使用、HTML変更なし。効果がビデオと同じ。
- マウスがボタン上にホバーすると、対応するコーナーから4つのアークが出現しサイズが拡大する。
- マウスがボタンから離れると、アークが元の位置に戻る。
- マウスがボタンから離れると、4つのカラーバーが左から右へ下から上へ移動して消える。
- 4つのカラーバーがボタン上のテキストを覆わない。

### A12: タイピング (Level 1)

提供されたビデオ `typing.mp4` に基づいてタイピングとテキスト削除のアニメーションを実装し、アニメーションをループさせる。

JSやCSSライブラリは使用不可。

`main.js` ファイルに使用する文字列の配列が含まれている。すべてのファイルを編集可能だが、`main.js` 内の配列の内容は変更しないこと。配列は以下の通り:

```js
const texts = ['David Smith', 'Web Developer', 'Web Designer', 'Competitor of Worldskills'];
```

要件ノート

- タイピングアニメーションと削除アニメーションが正しく動作すること。

### A14: リップルカード (Level 2)

- HTMLとCSSのみが使用されている。効果がビデオと同じ。

### A15: オリンピックゲームズ (Level 1)

CSSのみを使用して、提供されたサンプルと同じオリンピックゲームズの円を作成する。

色: Blue: `#0081C8`; Black: `#000`; Red: `#EE334E`; Yellow: `#FCB131`; Green: `#00A651`。

円は正確に100px、太さは10px。

要件ノート

- 5つの円すべてが存在する。
- 正しい色が使用されている (Blue: `#0081C8`; Black: `#000`; Red: `#EE334E`; Yellow: `#FCB131`; Green: `#00A651`)。
- プロポーションが維持されている。
- z-indexが維持されている (yellowがblueの上、blackがyellowとgreenの上、greenがredの上)。

### A16: 3Dフリップカード (Level 3)

CSSのみを使用 (JavaScriptは不可) して3Dフリップカード効果を作成する。提供されたフォントと色を使用し、提供されたサンプルの通りに実装すること。

Purple: `#4a0d66`; Blue: `#0e779f`。

カードはホバー時に表から裏へフリップし、3D効果のテキストを作成する。表面のボーダーと"Web Technologies"テキストは回転時に「フローティング要素」として表示される。裏面にはテキストとボタンが3D要素として表示される。

要求される効果は `media_sample/3d_flip_card_sample.mp4` に示されている。

要件ノート

- カードの表面が提供されたサンプルと類似している。
- カードの裏面が提供されたサンプルと類似している。
- カードの表面にボーダーとテキストの3D要素がある。
- カードの裏面にタイトルとボタンの3D要素がある。

### A17: シンプルキューブ回転 (Level 2)

以下の条件でシンプルなキューブ回転を実装する:

CSSのみ、JavaScriptは不可。

色: red: `#e1251b`; green: `#008c45`; yellow: `#f7e017`。色は全6面で対向方向に配置すること。

- Red: 前/後 (Communication and Interpersonal skills + Work organisation and management)
- Yellow: 左/右 (Design implementation + Speed)
- Green: 上/下 (Back-end Development + Front-end Development)

回転は"Work organisation and management"から開始し、各面で時計回り/上下の順で表示される。全6面が表示されること。

要求される効果は `media_sample/simple_cube_rotation_sample.mp4` に示されている。

要件ノート

- CSSのみでキューブが回転している (JSなし)。

    }
    return result;
}
```

```

提供されたHTMLテンプレートファイルを使用して、`data.json`のニュースデータに基づいた静的ウェブサイト（シングルページアプリケーションではない）を生成する。ニュースリストのページネーションを含み、1ページあたり5件のニュースを表示する。リスト内の各ニュースはtitle、excerpt、dateのみ表示する。ニュースアイテムをクリックするとニュース詳細ページにリダイレクトし、title、description、dateを表示する。

ニュースリストページのテンプレートファイルは `index.html`、ニュース詳細ページのテンプレートファイルは `news.html`。

ニュースリストページのページネーション機能では、現在のページのリンクがハイライトされること。最初のページではPreviousボタンが無効、最後のページではNextボタンが無効であること。

### C3: CSVコンバーター (Level 2)

`<path/to/some-name>.csv` ファイルへのパスをクエリパラメータとして受け取り、同じパスに `<path/to/some-name>.json` ファイルを返すスクリプトを作成する。CSVの内容（レコードのリスト）をJSON出力に変換する。CSVの最初の行は変換用に動的に考慮される列のリストを含む。レコード内の空の値は `""` として保存される。CSVとその結果のJSONの例が提供される。

要件ノート

- 入力パラメータと同じディレクトリに同じベースネームで出力（拡張子が異なる）。
- 動的な列数に対応。
- サンプルと同じまたは類似のデータ入力で期待される出力を生成。

### C4: ストリングリバーサー - PHP (Level 1)

PHPを使用して、HTMLフォームからのユーザー入力文字列（例: "worldskills"）を受け取り、文字列を反転し、大文字に変換し、ページリロードなしで同じページに結果を表示する。

要件ノート

- フォームがPOSTメソッドで入力文字列を正しく送信する。
- フォームとPHPコードが入力バリデーションを正しく処理し、空でない文字列のみ処理する。
- 入力された単語が大文字に変換される。
- 入力された単語が反転される。

### C5: リカーリングカレンダー (Level 3)

繰り返しカレンダーを作成する。スケジュールはデータベースで管理し、各スケジュール（繰り返しを含む）は1行のみとする。

Title、Task Date、Is recurringを選択でき、繰り返しが有効な場合のみType、Cycle、End Dateを入力できる。カレンダーには前月と翌月の一部が含まれることがある。

TypeはDay、Week、Month、Yearのいずれかで、Cycle数のType毎にスケジュールが繰り返される。

例: TypeがWeekでCycleが2の場合、スケジュールは2週間ごとに繰り返される。

繰り返しスケジュールの日付がその月の最終日を超える場合、その月の最終日に表示される。

ユーザーは表示されるカレンダーの年と月を変更できる。

要件ノート

- 日付変更フォーム、選択月のカレンダー、イベント追加フォームがある。
- 単一スケジュールを追加できる。
- 繰り返しスケジュールを追加できる。
- 単一スケジュールが正しく表示される。
- 繰り返しスケジュールが正しく表示される（繰り返しスケジュールの日がその月の最終日を超える場合は最終日に表示。タスク日以降の繰り返しのみカレンダーに表示）。
- カレンダーの日付を変更できる。

### C6: SQLデータインポート (Level 1)

コンソールゲームのデータを含むCSVファイル (`console_games.csv`) が提供される。データベース `data_import` を作成し、CSVファイルの全データをデータベースの `console_games` テーブルとしてインポートする。ソリューションを1つのSQLファイル `console_games.sql` として提供し、全データインポート後にデータベースのSQLダンプを `data_import_dump.sql` として提供する。

注意: ローカルホストで処理する場合、ローカルファイルのアップロードを許可するために `my.ini` を編集し、以下をファイルの末尾に追加してMySQLを再起動する必要がある場合がある。

```ini
#Enable local files upload
[mysqld]
secure_file_priv=""
```

ライブURLサーバーの場合は絶対パスが必要。

要件ノート

- 動作するソリューションがSQLファイルとして提供されている。
- `sql_import_dump.sql` が正しいネーミングで提供されている。
- データベース名が `data_import` として正しい。
- 全CSVデータがインポートされている（10列 / 最終ゲームID 15979）。

### C7: エクセルダウンロード (Level 3)

### C8: ユーザー登録 (Level 3)

2つのフォルダが提供され、PHPまたはNode.js + Expressのアプローチを選択できる。

レイアウト、ユーザーテーブル作成用SQLファイル、接続とデータベースリセット用の基本スクリプトが提供される。

ユーザーがユーザー名とメールアドレスで登録できるユーザー登録フォームを作成する。登録済みユーザーのリストと登録時間を表示する。メールアドレスが既に存在する場合、エラーメッセージ"Email is already registered"を返す。

要求される動作は提供された `media_sample/user_registration.mp4` に示されている。

要件ノート

- `register` ボタンが新しいエントリーをデータベースに追加し、テーブルに返す（データベースにのみ存在しフロントエンドにない場合はエラーとして扱う）。
- 新しいエントリーがページリフレッシュなしで `registered users` テーブルに表示される。
- 既存のメールアドレスは挿入できず、エラーメッセージが表示される。
- データベースが正しくリセットされる（リセット後にIDが1から開始しない場合などはエラーとして扱う）。

### C9: API CORS (Level 2)

フロントエンドアプリケーションとバックエンドアプリケーションが提供される。

バックエンドはVM Apacheサーバーを通じてアクセスし、フロントエンドアプリケーションには `npm run dev` を使用して、バックエンドとは異なるポートを使用する。

フロントエンドにアクセスすると、CORSにより4つのメソッドが正しく実行されない。

バックエンドアプリケーションの `cors.php` ファイルのみを変更して、すべてのリクエストが正しく実行されるようにする。

フロントエンドアプリケーション実行: `npm run dev` コマンドで開発サーバーを起動できる。

リクエストアドレス設定: フロントエンドの `index.html` を開き、`API_HOST` 変数をバックエンドに合わせて変更する必要がある。

(`node_modules` は圧縮ファイルで提供。`README.md` を必ず参照すること)

要件ノート

- 開発サーバーを開いて接続すると、バックエンドアプリケーションにリクエストが送信される。
- GETメソッドリクエストが正しく動作する。
- POSTメソッドリクエストが正しく動作する。
- PUTメソッドリクエストが正しく動作する。
- DELETEメソッドリクエストが正しく動作する。

### C10: リアルタイムチャット (Level 2)

フロントエンドとバックエンドに分かれたリアルタイムチャットアプリを作成する。

提供されたフロントエンドとバックエンドのテンプレートとsocketを使用して、現在接続しているユーザーにメッセージを送信できること。

自分のメッセージと他のユーザーからのメッセージを区別し、メッセージ送信後に入力フィールドをクリアすること。

(`node_modules` は圧縮ファイルで提供。`README.md` を必ず参照すること)

デプロイ

- バックエンドはテスト用に `npm run start` コマンドでサーバーを起動する。
- フロントエンドは指定されたURLにアクセスして動作すること。`/xx_module_a/realtime_chat`

要件ノート

- バックエンドとフロントエンドが別々に開発されている。
- フロントエンドが提供されたファイルを使用。
- バックエンドとフロントエンドが提供されたパッケージ `socket.io` を通じて通信する。
- フロントエンドでメッセージを入力すると、現在接続しているすべてのユーザーにメッセージが送信され、リアルタイムで表示される。
- メッセージ送信後、入力ウィンドウが空白になる。

### C11: イメージモザイク (Level 1)

提供された画像のモザイクを動的に作成する。

完成画像は正方形のセルで構成され、各セルは元の画像の対応するエリアの平均色値を持つ。

セルのサイズはクエリパラメータ `cell_size` で設定可能。`cell_size` が提供されない場合は50pxに設定すること。

要件ノート

- モザイクが `cell_size` クエリパラメータで指定されたサイズに従って作成される。
- `cell_size` クエリパラメータが提供されない場合、50pxのセルサイズでモザイクが作成される。
- モザイク画像が元の画像と同じサイズである。

### C12: ローカルテーブルデータコントロール (Level 1)

提供された `table.json` ファイルを読み込み、テーブル形式でユーザーに表示する。

ユーザーがテーブルデータ（フィールド、行）を変更し、行の追加・削除ができる。

追加・削除されたデータは直接 `table.json` に保存され、変更されたテーブルデータは保存ボタンを押して保存する。

要件ノート

- 提供されたJSONに基づいて正しいデータがユーザーに表示される。
- 削除または追加ボタンを押すと、変更がすぐにJSONに反映される。
- 保存ボタンを押すと、変更されたデータ（フィールドを含む）がJSONに反映される。
- 削除、追加、変更されたデータが既存のJSON構造に従って正しく保存される。

### C13: JSONファイルを使ったログインシステム (Level 3)

以下の仕様に基づいてPHPまたはNode.jsで機能を実装する。データベースは使用せず、JSONファイルにデータを保存する。JavaScriptは使用不可。

- ユーザー登録機能: HTMLフォームを作成。
- ユーザー情報の保存: ユーザーが送信したユーザー名とパスワードを `users.json` ファイルに保存する。保存前にパスワードをハッシュ化する。空のフィールドで登録しようとした場合にエラーメッセージを表示する。
- ログイン機能: ユーザーがユーザー名とパスワードを入力しサーバーに送信できるHTMLフォームを作成する。空のフィールドでログインしようとした場合にエラーメッセージを表示する。
- ユーザー認証: ユーザーが送信したユーザー名とパスワードを保存された情報と比較する。ユーザー名とハッシュ化されたパスワードが一致した場合、ユーザーを認証しログイン成功メッセージを表示する。一致しない場合はログイン失敗メッセージを表示する。
- ログアウト: ログイン完了時のみログアウトボタンを表示する。

要件ノート

- ユーザーを登録できる。
- 複数のユーザーを登録できる。
- ログインできる。
- ログアウトできる。
- 空のフィールド送信時にメッセージを表示する。
- UIがユーザーエクスペリエンスを考慮して調整されている。
